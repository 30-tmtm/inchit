# inchit 체크리스트 탭 레이아웃 요소 정리 (MVP)

본 문서는 베타 MVP 기준 체크리스트 탭 구성안을 정리합니다.

- 범위: 월령 고정 체크리스트 1개
- 기준: 영유아 발달정보 체크리스트는 `K-DST` 기준으로 구성
- 핵심 동작: 체크 토글
- 홈 연동: 홈 미니카드의 `보기` 버튼으로 체크리스트 탭 진입

### 타 탭과의 역할 구분(한 줄)

- 이 탭은 **K-DST 기반 발달 체크**와(및 여력 시) **상황별 규칙 기반 체크리스트**에 집중한다.
- `주간 시간표`와 `월간 캘린더`는 **서로 자동 연동되지 않는다.** **건강검진·예방접종** 로드맵(월령 구간·날짜 범위)은 **월간 탭의 보조 뷰(타임라인)** 에 두는 것을 기본으로 하고, 이 탭의 K-DST 체크와는 **목적·데이터를 분리**한다. (검진·접종을 체크리스트 “추가 템플릿”으로 넣는 것은 후순위로 검토)

## 1) 체크리스트 탭 레이아웃 구조 (위에서 아래 순서)

### 1. 상단 앱바
- 좌측: `체크리스트` 타이틀
- 우측(선택): `도움말` 또는 `정보` 아이콘 1개만 노출
- 원칙: 상단 액션 버튼을 최소화해 화면 피로도 감소

### 2. 월령 정보 헤더
- 현재 월령 라벨 (예: `월령 14개월`)
- 보조 문구: `현재 월령 기준 체크리스트입니다`
- 기준 표기: `K-DST 기준 항목`
- (선택) 마지막 갱신일 표기

### 3. 진행률 영역
- 진행률 텍스트: `2/5 완료`
- 간단 진행 바(선택)
- 안내 문구: `체크하면 완료 날짜가 기록돼요`
- 톤: 숙제 느낌을 줄이기 위해 숫자 강조보다 "가벼운 문장" 중심

### 4. 체크리스트 항목 리스트 (핵심)
- 섹션 형태:
  - 카드 1개 안에 그룹 타이틀 + 진행률(`0/6`) + 펼침/접힘
  - 기본값은 펼침(Expanded), 필요 시 접힘(Collapsed) 지원
- 리스트 아이템 구조:
  - 좌측: 체크박스 토글
  - 중앙: 항목명 텍스트
  - 우측(선택): 정렬 핸들(후순위) 또는 빈 상태 유지
- 최소 동작:
  - 체크 ON: 완료 상태 + 완료일 저장
  - 체크 OFF: 완료 상태 해제 (MVP에서는 당일 기록 기준 단순 처리)

### 5. 하단 보조 영역
- 저장 방식:
  - 기본: 토글 즉시 저장(별도 저장 버튼 없음)
- 보조 버튼(선택):
  - `완료 기록 보기` 또는 `다시 불러오기`
- FAB(선택):
  - 우하단 `+` 버튼 1개만 사용
  - 역할: 사용자 항목 추가(초기에는 비활성/준비중으로도 가능)

## 1-1) UI 톤 & 스타일 가이드 (레퍼런스 반영)
- 목표 감성: "가볍고 친근한 체크 경험", "숙제처럼 보이지 않는 화면"
- 시각 원칙:
  - 큰 여백 + 낮은 정보 밀도
  - 옅은 배경과 소프트 카드(강한 경계선 최소화)
  - 체크박스/텍스트 중심의 단순 리스트
  - 한 화면에 핵심 그룹 1~2개만 노출
- 문구 톤:
  - 격려형 헤드라인 사용 (예: `변화의 첫 걸음을 축하해요!`)
  - 지시형 문구/압박형 표현 지양
- 인터랙션 원칙:
  - 토글 즉시 반응
  - 애니메이션은 짧고 부드럽게(과한 효과 금지)

## 2) 항목 구성 기준 (K-DST)
- 항목 소스: `K-DST` 월령 구간별 발달 항목
- MVP 구성: 월령 구간 1세트만 노출
- 화면 표기 방식:
  - 항목명(사용자 친화 문장)
  - 체크 토글
  - 완료일
- 참고 문구(권장): `본 체크리스트는 K-DST 기준 참고용이며, 진단을 대체하지 않습니다.`

## 3) 상태별 표시 규칙

### 초기 빈 상태
- 문구: `현재 월령 체크리스트를 준비 중이에요`
- 버튼: `새로고침` 또는 `문의하기`

### 일부 완료 상태
- 완료 항목은 시각적으로 구분(텍스트 톤 다운/체크 강조)
- 진행률 실시간 반영
- 카드 상단에 `x/y` 진행률 고정 표시

### 전체 완료 상태
- 메시지: `오늘 체크리스트를 모두 완료했어요`
- 보조 액션: `내일 다시 보기`

## 4) MVP 데이터 규칙 (간단)
- 저장 필드 최소:
  - `child_id`
  - `checklist_item_id`
  - `kdst_age_group` (월령 구간)
  - `kdst_domain` (발달 영역, 선택)
  - `checked` (true/false)
  - `checked_at` (datetime, checked=true일 때)
- 기록 정책:
  - 베타 MVP에서는 복잡한 버전 관리 없이 최신 상태 우선 저장

## 4-1) K-DST 관련 운영/표기 원칙
- 신뢰성:
  - 체크리스트 상단 또는 정보 아이콘에서 `K-DST 기준`임을 명확히 표기
- 면책:
  - `참고용 정보`임을 안내하고, 의학적 판단/진단 대체 문구는 금지
- 확장(정식 런칭 이후):
  - 월령 구간 추가, 영역별 분류(대근육/소근육/언어/사회성 등) 고도화

## 5) 홈 미니카드 연동 규칙
- 홈의 `오늘 체크리스트 (K-DST 기준)` 미니카드에서 `보기` 클릭 시 체크리스트 탭으로 이동
- 홈에서는 체크 토글 미제공(진입만 제공)
- 홈 미니카드는 진행률만 노출 (`x/y 완료`)

## 6) 접근성/사용성 기본 원칙
- 체크 영역 터치 크기 충분히 확보(오입력 방지)
- 텍스트 대비 유지(색상만으로 상태 구분하지 않기)
- 스크롤 시 헤더 고정은 선택(항목 수가 많아질 때 고려)
- 한 화면에 과도한 버튼 노출 금지(핵심 CTA 1~2개)
